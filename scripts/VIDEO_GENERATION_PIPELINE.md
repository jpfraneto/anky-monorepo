# Video Generation Pipeline

Master reference for the daily bedtime-story video pipeline behind **Anky** on YouTube.

Paired with: `scripts/gods_pipeline.py`, `templates/station.html`, `src/routes/mod.rs` (`/api/station/*`).

---

## What this pipeline produces

For each daily episode:

- **4 final videos**, 1080×1920 vertical mp4 (AV1 + AAC):
  - `{god}_long_en.mp4` — ~8 minute full story, English
  - `{god}_short_en.mp4` — ~88 second trailer, English
  - `{god}_long_es.mp4` — ~8 minute full story, Spanish (neutral Latin American)
  - `{god}_short_es.mp4` — ~88 second trailer, Spanish
- **25 still images** (10 long-form + 15 short-form), 768×1344, generated by ComfyUI + Flux.1-dev + `anky_flux_lora_v2`
- **4 assembled multi-voice audio mp3s** (one per language × version)
- **2 per-language markdown scripts** with parent notes suitable for YouTube descriptions
- **Per-platform captions** (Instagram, TikTok, X, Farcaster, YouTube) in both languages
- **Telegram digest** to JP (`@ankydotappbot` → chat 5414944240)

Total cost per episode: ~$0.40 ElevenLabs + ~$0.02 LLM + ~$0.0007 Solana. Wall time: 35–50 minutes (dominated by image generation).

Channel strategy: **one channel called "Anky"**, one playlist per language.

## The frame — this is load-bearing

Every prompt, voice decision, and image choice descends from this. If you change anything in the pipeline, check against these.

### Audience
**4-year-old children** and the adults watching with them. Tonal reference: **Bluey**, not Paw Patrol. Pedagogical reference: **Rudolf Steiner / Waldorf early childhood** (the 0–7 window). The foundational message: the world is good, the world is beautiful, the world is true.

### Who Anky is in the story
Anky is **not a character in the story.** Anky is the name we give to the interior awareness — the consciousness that lives inside every alive being. When Anky speaks, the voice coming out of the speaker is the listening kid's own inner awareness, softened and given language. The kid could almost mistake Anky for themselves.

### Three roles per episode
- **The kid** — has a body, has feet, has a beating heart. Moves through the world, watches, sits. **Never speaks out loud.** The kid's experience comes through Anky's narration.
- **The god** — the only external presence with a body. Ancient, patient, gentle, genderless (in every language). Has 3–6 spoken beats per story. **The god does not teach.** The god honours.
- **Anky** — the interior narrator. **No body in the scene.** No hands, feet, hair, walking. Once per episode, marked with `[softly]` or `[whispers]`, Anky briefly becomes distinct and names itself ("I am here with you"). This is the Mode (b) moment. Everywhere else, Anky narrates in second person from the kid's awareness: "you are sitting," "the stone is warm."

### What the stories DON'T do
- No didactic lessons (no "the ground is not for comfort")
- No narrative arc in the screenwriting sense — small noticing, not dramatic rise/fall
- No scary imagery as plot device
- No kid dialogue
- No abstract vocabulary a 4-year-old doesn't use
- No more than 2–3 audio tags per story — tags are sacred, not sprinkled

### What the stories DO
- Emotion as weather, not topic (the kid feels warmth in their chest, not "the feeling of warmth")
- Repetition as rhythm ("the flower is very red. the flower is very quiet.")
- Reverence for small things
- Silence written in via line breaks and `...`
- The kingdom's element and kingdom's chakra are shown, never named

## Script format — stage play

Every generated script is in stage-play format with uppercase speaker labels:

```
ANKY: You are sitting on a warm stone.
      The sea is quiet today.
      A red flower is near your foot.

SHIVA: Hello, little one.
       I have been here a long time.

ANKY: [softly] Something warm in your chest.
```

Rules the generation prompt enforces:

- Every block begins with `ANKY:` or `<GOD_NAME>:` (uppercase, colon, text)
- Blank line separates blocks
- Consecutive lines by the same speaker stay in one block
- Audio tags are inline at the start of the line they colour: `[softly]`, `[whispers]`, `[warm]`, `[gently]`
- Opening (`{anky_greeting}`) and closing (`{closing}`) lines are untagged
- Use `...` for breath pauses

The parser (`parse_stage_play()` in `step_voice`) splits the script into a list of `{speaker, text}` segments. Unknown speaker labels get routed to `anky` so no content is ever dropped.

## The 8 pipeline steps

All run from `scripts/gods_pipeline.py`. Outputs land in `videos/gods/pipeline/*.md`. Final assets in `videos/gods/{god}/`.

### 1. Zeitgeist — `01_zeitgeist.md`
Four agents read "what's alive in humanity today":
- **Grok** (xAI API): reads X/Twitter pulse
- **Claude** (Anthropic): reads archetypes + deeper patterns — falls back to Qwen on 529
- **Qwen** (local, port 8080): independent wildcard
- **Anky**: Solana on-chain activity + enclave insights (enclave call is a 3s health check first, then proceeds only if up)

Slowest step, ~4–5 min. All four LLM calls run sequentially.

### 2. Council — `02_council.md`
Qwen picks `GOD`, `KINGDOM`, `CITY`, `WHY`, `ANGLE`, `EMOTIONAL CORE`, `LESSON` based on the zeitgeist. ~30s.

The 8 kingdoms × 3 cities × 10 gods live in the `KINGDOMS` dict and `VOICES` dict (see below). Override with `--god Shiva` CLI flag to force a specific god.

### 3. Script — `03_script_en.md`, `03_script_es.md`, `03_script.md` (index)
Per language, three LLM calls:
1. Full story (Bluey-paced, 500–700 words, stage-play format)
2. 88-second short (110–140 words, stage-play format, one Mode-b interior moment)
3. Parent note (150–200 words, YouTube-description ready)

The three prompts (`_story_prompt`, `_short_prompt`, `_parent_prompt`) carry the full frame — Steiner, Waldorf, Bluey register explicit in the system instructions. Language-specific guidance (`_language_guidance`) handles the Spanish gender-neutral god problem with `el dios` + reformulation to avoid `él/ella`.

~2–4 min total.

### 4. Prompts — `04_prompts.md`, `prompts.json`
Qwen generates image prompts. Each prompt is a `{type, prompt}` object where type ∈ {`OPENING`, `SCENE`, `INTERIOR`, `CLOSING`}:

- **OPENING** (1 per list): Anky iconography — blue skin, purple hair, golden eyes, symbolic
- **SCENE** (most prompts): kid + god + world. Anky is ABSENT from the frame. Subtle hints (glow in chest, gold in eyes) allowed in 1–2 frames.
- **INTERIOR** (1 per list, around prompts 6–8 long / 9–12 short): symbolic visualization of the Mode-b moment — eye with galaxy inside, ribs with glowing presence
- **CLOSING** (1 per list): mirror of opening, Anky going quiet

Long list: 10 prompts. Short list: 15 prompts. ~30s.

### 5. Images — `05_images.md`
For each prompt, queues ComfyUI workflow:
- UNet: `flux1-dev.safetensors`
- LoRA: `anky_flux_lora_v2.safetensors` @ 0.85 strength (both model + clip)
- VAE: `ae.safetensors`
- Dual CLIP: `clip_l.safetensors` + `t5xxl_fp8_e4m3fn.safetensors` (flux mode)
- Sampler: Euler, 20 steps, CFG 3.5, 768×1344 portrait

Polls `/history/{prompt_id}` every 2s up to 4 min. Saves to `videos/gods/{god}/long_scene_NNN.png` and `short_scene_NNN.png`. The markdown embeds every image inline via `![](/api/station/asset/{god}/{name}.png)`.

**~5–7 min** for 25 images on one RTX 4090 (GPU 1).

### 6. Voice — `06_voice.md`
The multi-voice magic. For each language and each version (long + short):

1. Parse the stage-play script into `[{speaker, text}]` segments
2. For each segment, call ElevenLabs v3 with that speaker's voice ID (see cast below)
3. Use `tts_v3_chunk()` with stability 0.5 (natural) for long, 0.3 (creative) for short
4. Concatenate all segment audio with **220ms silent pauses** between speaker shifts (`concat_with_pauses()`)
5. Write `voice_{full|short}_{code}.mp3`

Model: `eleven_v3` throughout. Max 5000 chars per API call (v3 limit); `chunk_by_paragraph()` splits if a single segment overflows (rare at our story length).

Karaoke timestamps are **not** produced — v3's `/with-timestamps` endpoint is v2-only. If we want word-level subtitles in the future, either run Whisper forced alignment locally, or do an extra v2 request on the tag-stripped text.

~3–4 min for all 4 audio outputs. Burns ~$0.30.

### 7. Video — `07_video.md`
For each (language × version), assembles mp4:
1. Probes audio duration with `ffprobe`
2. Calculates per-image duration = audio_duration / num_scenes
3. Builds concat file: each image shown for that duration, last frame held
4. Generates karaoke ASS subtitles if timestamps exist (currently only for short form in future v2 alignment case)
5. Runs `ffmpeg` with:
   - `scale=1080:1920:force_original_aspect_ratio=decrease,pad=1080:1920:(ow-iw)/2:(oh-ih)/2:black,format=yuv420p`
   - `-c:v av1_nvenc -cq 30 -preset p4` (GPU encode)
   - `-c:a aac -b:a 192k`
   - `+faststart` for web streaming
6. Writes `{god}_{long|short}_{code}.mp4` to `videos/gods/{god}/`

~2–3 min for all 4 videos on NVENC.

### 8. Publish — `08_publish.md`
Per language:
1. Extract parent note from `03_script_{code}.md`
2. Extract short script preview
3. Qwen generates per-platform captions (Instagram, TikTok_title, TikTok_desc, X_tweet, Farcaster, YouTube_title)
4. Parent note becomes the YouTube description
5. Compose dual-language Telegram digest (copy-paste captions + file paths)
6. Send via `notify_jp_telegram()` → Hermes bridge or direct Telegram API

~30s–1 min.

## Audio tag policy (ElevenLabs v3)

Tags in square brackets, placed inline before the sentence they colour.

**Budget:** 2–3 per full story. 1–2 per short. Tags are sacred; the default register is already tender. Most lines have no tag.

**Allowed tags:**
- **Mode-b interior moment** (mandatory, once per episode): `[softly]` or `[whispers]`
- **Optional gentle colouring**: `[warm]`, `[gently]`, `[curious]` — max one per episode beyond the interior moment
- **Breath pauses**: use `...` (ellipsis) in text rather than tags

**Banned:**
- Action tags (`[walks]`, `[hits drum]`)
- Stacked tags on one line
- Tags on opening greeting or closing line

**v3 stability mapping** (the station API translates strings → numeric):
- `creative` = 0.3 (expressive, tags on strongly) — used for short form
- `natural` = 0.5 (balanced) — used for full story
- `robust` = 0.75 (ignores tags) — not used

## Station dashboard

**URL:** `http://localhost:8889/station`

### What you see
- Left sidebar: 8 steps, green when complete
- Main area: renders the selected step's markdown. Images and `<audio>`/`<video>` tags are embedded inline.
- Script step shows **EN / ES language tabs** when both files exist.
- Any pending/running step shows a **live log tail** auto-refreshing every 2s with the `● RUNNING` / `○ WAITING` badge. When the step's `.md` lands, the view flips to rendered content.
- Right panel: Voice playground + video preview + log viewer tabs.

### Endpoints (`src/routes/mod.rs`)
- `GET /api/station/steps` — lists `.md` files in pipeline dir with sizes
- `GET /api/station/step/{name}` — reads one file (including `pipeline.log`)
- `POST /api/station/run` — spawns full pipeline in background
- `POST /api/station/run/{step}` — spawns single step via `--step {step}`
- `POST /api/station/tts` — ElevenLabs proxy for the playground. Accepts `{text, voice_id, model_id, stability}`. Maps named stability presets to numeric. Logs real error body on failure.
- `GET /api/station/asset/{*rel}` — serves files from `videos/gods/` (images, audio, video, subtitles). Path traversal blocked.

### Voice playground
Dropdown has all EN + ES cast voices + backups. Also: model selector (v3 / multilingual v2 / turbo v2.5), stability preset (creative / natural / robust), **audio-tag chips** — click to insert `[whispers]` etc. at cursor.

## Running the pipeline

### Full run
```bash
# CLI
cd ~/anky && python3 scripts/gods_pipeline.py

# CLI with god override
cd ~/anky && python3 scripts/gods_pipeline.py --god Shiva

# Web (from anywhere)
curl -X POST http://localhost:8889/api/station/run
# or click GENERATE TODAY'S EPISODE in the station UI
```

Warning: full run **clears all `*.md`, `*.json`, `*.log` from the pipeline dir** at startup. Finished episode assets in `videos/gods/{god}/` are preserved.

### Single step
```bash
python3 scripts/gods_pipeline.py --step script
# steps: zeitgeist | council | script | prompts | images | voice | video | publish

# Or via HTTP
curl -X POST http://localhost:8889/api/station/run/script
```

Each step reads its upstream `.md` files and writes its own. Most steps are safely idempotent except step 1 (new zeitgeist each time).

### Avoiding duplicates
The web endpoints don't debounce. If you click GENERATE twice, you get two pipelines stomping on each other. Kill with `pkill -9 -f gods_pipeline.py`.

## Known failure modes

- **OOM kill of Qwen** — system memory pressure during image generation (Flux holds ~24GB VRAM + CPU RAM spikes). The pipeline survives if Qwen restarts between steps. The frontend browser tabs are the usual culprit for memory pressure.
- **Anthropic 529** — transient overload, very common. Always falls back to Qwen. No action needed.
- **Enclave down** — logged and skipped cleanly. Doesn't block the pipeline.
- **Duplicate runs** — clicking GENERATE multiple times spawns parallel pipelines. Each tries to write to the same pipeline dir. Detect with `ps aux | grep gods_pipeline.py`. Kill all with `pkill -9 -f gods_pipeline.py`.
- **Stage-play parser sees unknown speaker** — routes to Anky and logs a warning. Check the script for LLM-invented speakers other than `ANKY:` and the god's uppercase name.
- **Segment over 4800 chars** — auto-chunked by paragraph. Rare at current story lengths.
- **v3 stability as string** — the ElevenLabs API rejects string presets. The station handler maps them to numeric. If you call the API directly from python, pass 0.3 / 0.5 / 0.75.

## Tuning knobs

- **Story length**: `_story_prompt` target is 500–700 words (~7–8 min at Bluey pace with pauses + silence). Change in the LENGTH block of the prompt.
- **Short length**: 110–140 words target.
- **Number of images**: 10 long + 15 short, set in step 4's `DISTRIBUTION` block and again in step 5 ordering. Images per minute controlled by audio duration / scene count.
- **Audio pause between speaker shifts**: 220ms in `concat_with_pauses()`.
- **v3 stability per version**: 0.5 for long (`VOICE_MODELS["full"]`), 0.3 for short (`VOICE_MODELS["short"]`).
- **ComfyUI sampling**: 20 steps, CFG 3.5, Euler — in the workflow in `step_images`.
- **LoRA strength**: 0.85 both model + clip. Lower for less Anky-style bleed-through into SCENE prompts.

## Extending the pipeline

### Add a new god
1. Pick the god. Assign a kingdom + city in `KINGDOMS`.
2. Add to `VOICES` (English premade voice ID) and `VOICES_ES` (Latin American Spanish community voice).
3. Update the station dropdown in `templates/station.html`.
4. No other code changes — the parser and voice router pick up new god names automatically via `_voice_for()` lookup by god name lowercased.

### Add a new language
1. Add to `LANGUAGES` list in `gods_pipeline.py` with `code`, `name`, `anky_greeting`, `closing`, `full_cta`.
2. Add `_language_guidance(lang)` case with grammar notes (especially the genderless-god handling for gendered languages).
3. Add a `VOICES_{CODE}` dict with at least `anky` and one entry per god. Pipeline reads it via `_voice_for` with a dict-naming convention — update the helper if adding more than EN + ES.
4. Add native voices for the god cast (community library).

### Add a new step
Insert a function `step_foo()`, add it to the `STEPS` dict, append to `run_full_pipeline()` in order. Make sure it reads its upstream file and writes a `NN_foo.md`. The station picks it up automatically if `NN_foo.md` is listed in the `STEP_NAMES` array in `station.html`.

## What's intentionally not built yet

- **Karaoke subtitles** on the long form — requires Whisper forced alignment locally (v3 doesn't give timestamps, v2 doesn't handle tags, long form is too expensive to double-bill). Decision: skip for now. Add if YouTube Shorts analytics show low retention at non-captioned points.
- **Kid voice** — intentionally absent. The frame is that the listener IS the kid. A separate kid voice creates a character between the listener and Anky, breaking identification. See conversation log around 2026-04-13 for the full argument.
- **Automated platform upload** — Telegram digest gives JP everything to paste. TikTok/Instagram/YouTube API posting is a follow-up.
- **Per-scene Ken Burns on long form** — current assembly uses static images with duration = audio_length / scene_count. A pan/zoom filter per scene would add texture.
- **Music bed** — no background music currently. ElevenLabs also has a music generation API if we want to try.
- **Sound effects** — wave crash, drum, wind. Could come from ElevenLabs sound effects API or a library. Would need cue points embedded in the script format.
